Shader Class

A base class for custom shaders, providing basic shader compilation, linking and reflection facilities.

Syntax

public abstract class Shader : IDisposable

Remarks

Shaders are generally used by subtypes of GameStack.Graphics.Material and shaders that take custom parameters often have their own material types exposing those parameters as well.

Shader Constructor

Syntax

public Shader (string vertSource, string fragSource, string name)

Parameters

vertSource
The vertex shader GLSL source code.
fragSource
The fragment shader GLSL source code.
name
The unique name of the shader instance, used for reference tracking.

Remarks

The shader name is used by the reference counting mechanism to determine if two shaders of the same type are in fact identical shaders. One might construct two shaders of the same type but with different parameters, affecting how the shader code is generated. For statically defined shaders, using the default (the type name) is sufficient. Dynamically generated shaders must supply unique names based on any arguments affecting generation of shader code. Failure to do so may result in shader instances being reused when they should not be. Take note and heed this warning before ye descend into madness.
